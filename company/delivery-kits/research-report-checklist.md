# Research Report Delivery Kit

Last updated: 2026-03-10
Owner: CTO

## Purpose

Turn research work into a decision-ready output instead of an information dump.

## Definition Of Ready

Do not start until these are clear:
- order ID
- research question
- decision the report should support
- geography, market, or comparison scope
- must-include competitors or sources
- payment state
- deadline

## Start Checklist

- confirm the decision the client needs to make
- reduce scope to a clear comparison set or question list
- state what the report will not cover
- open the order row in the dashboard

## Research Checklist

- prefer recent and primary sources where available
- separate facts from inferences in notes
- track source links while researching
- stop collecting background once the decision question can be answered
- focus on implications, not just summaries

## Report Checklist

- executive summary answers the question directly
- findings section is evidence-based
- recommendations are specific and prioritized
- assumptions and uncertainties are visible
- next actions are practical for the client

## Delivery Package

Send:
- final report
- source list if included in scope
- revision window and scope reminder

## Internal Reuse Capture

After delivery, record at least one:
- source set worth reusing
- clearer framing question for future sales
- recurring decision pattern
- anonymized proof asset or case-study angle
