# Delivery Quality Checklist

Last updated: 2026-03-10
Owner: CTO

## Purpose

Prevent rushed delivery from turning into rework, refunds, or weak proof assets.

## Universal Checks

Complete before every delivery:
- scope matches the quote
- payment state allows delivery
- files are named clearly
- obvious errors are removed
- revision boundaries are restated
- next step for the client is clear

## Offer-Specific Checks

### Content Creation

- draft matches requested audience and outcome
- structure is readable and complete
- claims and references are checked where needed
- tone matches the brief
- CTA or intended use is clear

### Code Review

- findings are prioritized by severity
- references point to concrete files or code areas
- recommendations are actionable
- major risks are separated from minor improvements
- report avoids filler and generic advice

### Research Report

- question is answered directly
- sources are recent and credible
- findings are separated from inference
- recommendation section is specific
- report gives a decision, not just background

## Definition Of Done

An order is not done until:
- delivery is sent
- dashboard status is updated
- revision window is clear
- proof asset request is queued or sent
- one reusable internal asset is captured
