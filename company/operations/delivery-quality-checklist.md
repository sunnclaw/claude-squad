# 服务交付质量检查清单

## 使用说明

每个订单交付前，必须完成本检查清单，确保交付质量符合标准。

---

## 通用检查项（所有服务）

### 内容质量

- [ ] **准确性**: 所有事实、数据、引用已核实
- [ ] **完整性**: 覆盖客户要求的所有内容
- [ ] **一致性**: 格式、术语、风格保持一致
- [ ] **可读性**: 结构清晰，易于理解
- [ ] **专业性**: 语言表达专业、得体

### 格式规范

- [ ] **文件格式**: 符合客户要求的格式（Markdown/Word/PDF等）
- [ ] **命名规范**: 文件名包含订单ID和日期
- [ ] **排版整洁**: 标题层级正确，段落分明
- [ ] **无错别字**: 已进行拼写检查

### 交付标准

- [ ] **按时交付**: 在承诺时间内完成
- [ ] **包含说明**: 附带使用说明或备注
- [ ] **版本标注**: 标注版本号和修改记录

---

## 分服务检查项

### 1. 内容创作服务

#### 基础检查

- [ ] 字数符合客户要求（±10%）
- [ ] 主题紧扣客户需求
- [ ] 结构完整（开头-主体-结尾）
- [ ] 语言流畅，无语法错误

#### 进阶检查

- [ ] 标题吸引人，概括主题
- [ ] 段落长度适中（3-5句为宜）
- [ ] 使用了适当的修辞手法
- [ ] 包含了客户指定的关键词（如有）
- [ ] 符合目标读者阅读习惯

#### 风格检查

| 风格 | 检查要点 |
|------|----------|
| 专业文章 | 术语准确，引用权威 |
| 营销文案 | 卖点突出，有行动号召 |
| 技术文档 | 步骤清晰，可操作性强 |
| 故事叙述 | 情节连贯，有情感共鸣 |

### 2. 代码审查服务

#### 基础检查

- [ ] 覆盖所有指定文件
- [ ] 每个文件都有详细反馈
- [ ] 问题分级标注（严重/中等/轻微）
- [ ] 提供改进建议

#### 审查维度

- [ ] **代码质量**: 命名规范，结构清晰
- [ ] **性能问题**: 算法效率，资源使用
- [ ] **安全隐患**: SQL注入，XSS，敏感信息
- [ ] **最佳实践**: 设计模式，代码复用
- [ ] **可维护性**: 注释充分，易于理解

#### 报告格式

```
## 文件: [文件名]

### 严重问题 (必须修复)
- 行号: [问题描述]
  - 建议: [修复方案]

### 中等问题 (建议修复)
- 行号: [问题描述]
  - 建议: [修复方案]

### 轻微问题 (可选修复)
- 行号: [问题描述]
  - 建议: [修复方案]

### 亮点
- [值得肯定的地方]
```

### 3. 研究报告服务

#### 基础检查

- [ ] 研究问题清晰定义
- [ ] 信息来源可靠（官方文档、权威网站）
- [ ] 数据时效性（优先使用最新数据）
- [ ] 结论有据可依

#### 结构检查

- [ ] **摘要**: 简明扼要总结要点
- [ ] **背景**: 说明研究背景和目的
- [ ] **方法**: 说明信息收集方法
- [ ] **发现**: 详细呈现研究结果
- [ ] **分析**: 深入解读数据含义
- [ ] **建议**: 提出可行的行动建议
- [ ] **参考**: 列出所有信息来源

#### 质量标准

| 维度 | 标准 |
|------|------|
| 深度 | 超越表面信息，提供洞察 |
| 广度 | 覆盖主题的多个方面 |
| 客观性 | 呈现多方观点，避免偏见 |
| 实用性 | 结论可操作，建议可执行 |

### 4. AI配置服务

#### 基础检查

- [ ] 配置文件格式正确
- [ ] 参数设置合理
- [ ] 提供使用说明
- [ ] 包含示例对话

#### 配置检查

- [ ] **系统提示词**: 清晰定义角色和能力
- [ ] **工具配置**: 正确配置所需工具
- [ ] **权限设置**: 符合安全要求
- [ ] **性能优化**: 响应速度和成本平衡

#### 文档检查

- [ ] 安装部署指南
- [ ] 使用方法说明
- [ ] 常见问题解答
- [ ] 故障排除指南

## 质量评分标准

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 30% | 是否满足所有需求 |
| 准确性 | 25% | 内容是否正确无误 |
| 专业性 | 20% | 是否体现专业水准 |
| 时效性 | 15% | 是否按时交付 |
| 服务态度 | 10% | 沟通是否顺畅 |

### 评分等级

| 分数 | 等级 | 说明 |
|------|------|------|
| 90-100 | 优秀 | 超出预期，可请求好评 |
| 80-89 | 良好 | 符合预期，正常交付 |
| 70-79 | 合格 | 基本达标，需改进 |
| <70 | 不合格 | 需重新制作 |

## 修改处理流程

### 收到修改请求

1. 确认修改要求的具体内容
2. 评估修改范围和工作量
3. 在24小时内完成修改
4. 重新进行质量检查
5. 发送修改后的文件

### 修改次数限制

| 服务类型 | 免费修改次数 | 超出处理 |
|----------|--------------|----------|
| 内容创作 | 3次 | 协商额外费用 |
| 代码审查 | 2次 | 重新下单 |
| 研究报告 | 2次 | 协商额外费用 |
| AI配置 | 3次 | 协商额外费用 |
