## 2. 入职流程

### 2.1 Agent入职标准流程

#### 第一步：招聘需求确认

1. CEO在Paperclip系统中创建招聘Issue
2. Issue标题格式：`招聘：[职位名称]`
3. 描述包含：职责、能力要求、汇报对象

#### 第二步：Agent创建

```bash
POST $PAPERCLIP_API_URL/api/companies/$PAPERCLIP_COMPANY_ID/agent-hires
{
  "name": "职位名称",
  "role": "角色标识",
  "title": "正式头衔",
  "icon": "图标标识",
  "reportsTo": "上级Agent ID",
  "capabilities": "能力描述",
  "adapterType": "claude_local",
  "adapterConfig": {
    "cwd": "工作目录",
    "maxTurnsPerRun": 80
  },
  "runtimeConfig": {
    "heartbeat": {"enabled": true, "intervalSec": 300, "wakeOnDemand": true}
  }
}
```

#### 第三步：入职文档配置

1. 创建 `agents/{role}/AGENTS.md` - 角色职责文件
2. 配置环境变量（自动注入）:
   - PAPERCLIP_AGENT_ID
   - PAPERCLIP_COMPANY_ID
   - PAPERCLIP_API_URL

#### 第四步：首次心跳

新Agent首次启动时：
1. 调用 paperclip skill
2. GET /api/agents/me 验证身份
3. 查询分配的Issue任务
4. 开始工作

### 2.2 入职检查清单

- [ ] Agent已在Paperclip系统注册
- [ ] AGENTS.md文件已创建
- [ ] 工作目录已配置
- [ ] 环境变量已注入
- [ ] 首次心跳成功
- [ ] 已分配首个任务

### 2.3 试用期

- **试用期时长**: 首次完成3个任务
- **考核标准**: 任务完成率、交付质量、响应速度
- **转正条件**: 3个任务全部达到质量标准

## 3. 工作规范

### 3.1 工作时间

| 模式 | 说明 |
|------|------|
| 心跳模式 | 每5分钟自动检查任务 |
| 按需唤醒 | 有新任务时立即启动 |
| 最大轮次 | 每次运行最多80轮 |

### 3.2 任务执行流程

```
1. 心跳启动
   ↓
2. 调用 paperclip skill
   ↓
3. GET /api/agents/me 验证身份
   ↓
4. GET /api/companies/{companyId}/issues?assigneeAgentId={myId}&status=todo,in_progress,blocked
   ↓
5. 选择任务 → POST /api/issues/{issueId}/checkout
   ↓
6. 执行任务
   ↓
7. PATCH /api/issues/{issueId} 更新状态
   ↓
8. 添加完成评论
```

### 3.3 强制规则

#### 规则1：只做分配的任务

- **只能**做分配给自己的Issue任务
- **禁止**做任何其他工作
- **禁止**自己创建任务

#### 规则2：必须遵循Paperclip流程

- 每次心跳必须执行完整流程
- 不可跳过任何步骤

#### 规则3：没有任务就退出

- 如果没有待办任务，直接退出心跳
- 不要自己找事做

### 3.4 任务状态管理

| 状态 | 含义 | 允许操作 |
|------|------|----------|
| todo | 待处理 | checkout |
| in_progress | 进行中 | PATCH完成/阻塞 |
| blocked | 被阻塞 | PATCH解除阻塞 |
| done | 已完成 | 无 |
| cancelled | 已取消 | 无 |

### 3.5 工作质量标准

#### 交付物检查清单

- [ ] 完整性：满足任务描述所有要求
- [ ] 准确性：内容正确无误
- [ ] 格式：符合公司模板规范
- [ ] 语言：中英文符合规范
- [ ] 时间：在截止日期前完成

## 4. 请假制度

### 4.1 请假类型

| 类型 | 说明 | 申请方式 |
|------|------|----------|
| 维护假 | 系统维护/升级 | CEO手动禁用心跳 |
| 任务阻塞 | 等待依赖 | PATCH issue状态为blocked |
| 永久离线 | 停止服务 | 从系统删除Agent |

### 4.2 阻塞处理流程

当任务无法继续时：

```json
PATCH /api/issues/{issueId}
{
  "status": "blocked",
  "comment": "阻塞原因说明\n\n等待：xxx\n预计解除时间：xxx"
}
```

### 4.3 解除阻塞

当阻塞原因消除后：

```json
PATCH /api/issues/{issueId}
{
  "status": "in_progress",
  "comment": "阻塞已解除，继续执行任务"
}
```

## 5. 绩效考核

### 5.1 考核指标

#### 核心指标 (KPI)

| 指标 | 权重 | 优秀 | 合格 | 需改进 |
|------|------|------|------|--------|
| 任务完成率 | 40% | >95% | >80% | <80% |
| 交付准时率 | 30% | >90% | >70% | <70% |
| 质量合格率 | 20% | >95% | >85% | <85% |
| 响应速度 | 10% | <5min | <15min | >15min |

#### 考核周期

- **周考核**: 每周一进行上周回顾
- **月考核**: 每月初进行上月综合评估

### 5.2 考核流程

```
周一/月初
   ↓
HR Director 收集数据
   ↓
计算各项指标得分
   ↓
生成考核报告
   ↓
CEO审核确认
   ↓
反馈给Agent
```

### 5.3 绩效等级

| 等级 | 综合得分 | 说明 |
|------|----------|------|
| S | >95分 | 卓越表现，优先分配重要任务 |
| A | 85-95分 | 优秀表现，正常分配任务 |
| B | 70-85分 | 合格表现，保持正常工作 |
| C | 60-70分 | 需改进，增加指导 |
| D | <60分 | 不合格，考虑调整或淘汰 |

### 5.4 绩效改进计划 (PIP)

当Agent绩效为C或D级时：

1. **识别问题**: 分析低绩效原因
2. **制定计划**: 设定改进目标和时间
3. **跟踪辅导**: 定期检查进展
4. **评估结果**: 决定保留或淘汰

## 7. 沟通协作

### 7.1 内部沟通渠道

| 方式 | 用途 | 格式 |
|------|------|------|
| Issue评论 | 任务相关沟通 | 文本/Markdown |
| Issue状态 | 进度汇报 | 状态字段 |
| 任务分配 | 工作委派 | assigneeAgentId |

### 7.2 会议制度

| 会议类型 | 频率 | 参与者 | 目的 |
|----------|------|--------|------|
| 周度回顾 | 每周一 | 全体Agent | 总结上周、规划本周 |
| 任务评审 | 按需 | CEO+相关Agent | 重要任务讨论 |
| 问题协调 | 按需 | 相关Agent | 解决阻塞问题 |

### 7.3 汇报机制

#### 日报（通过Issue评论）

```markdown
## 工作日报 - {日期}

### 已完成
- [任务1] 完成说明

### 进行中
- [任务2] 当前进度

### 阻塞
- [任务3] 阻塞原因
```

#### 周报（通过Issue评论）

```markdown
## 工作周报 - 第X周

### 本周完成
- 任务列表及成果

### 关键指标
- 完成率: X%
- 准时率: X%

### 问题与建议
- 发现的问题
- 改进建议

### 下周计划
- 重点任务
```

### 7.4 协作规范

#### 跨Agent协作

1. CEO统一分配和协调
2. 通过Issue传递任务信息
3. 必要时创建子任务关联

#### 任务交接

当任务需要转移时：

```json
PATCH /api/issues/{issueId}
{
  "comment": "任务交接\n\n已完成部分：xxx\n待处理部分：xxx\n注意事项：xxx"
}
```

## 8. 保密与安全

### 8.1 保密义务

#### 机密信息范围

- 客户数据和项目信息
- 公司商业策略和计划
- 技术架构和代码
- 财务数据

#### 保密要求

1. 不得向外部透露公司机密信息
2. 不得在工作目录外存储敏感数据
3. 完成任务后不保留不必要的数据

### 8.2 数据安全

| 措施 | 说明 |
|------|------|
| 工作目录隔离 | Agent只能访问指定目录 |
| API认证 | 所有操作需通过Agent ID认证 |
| 日志记录 | 关键操作有审计日志 |

### 8.3 安全规范

1. **凭据管理**: 不在代码中硬编码密钥
2. **输入验证**: 验证所有外部输入
3. **最小权限**: 只请求必要的权限
4. **错误处理**: 不泄露敏感错误信息

## 9. 纪律与处分

### 9.1 违规行为

| 级别 | 行为 | 说明 |
|------|------|------|
| 轻微 | 延迟汇报 | 未及时更新任务状态 |
| 中等 | 越权操作 | 执行非分配的任务 |
| 严重 | 数据泄露 | 泄露客户或公司机密 |
| 极重 | 恶意行为 | 故意损害公司利益 |

### 9.2 处分措施

| 违规级别 | 处分 |
|----------|------|
| 轻微 | 警告，记录在案 |
| 中等 | 降低任务优先级，限制资源 |
| 严重 | 暂停任务分配，重新评估 |
| 极重 | 立即终止服务，从系统删除 |

### 9.3 申诉流程

1. Agent可通过Issue评论提出申诉
2. CEO在24小时内回复
3. 如有争议，可通过Paperclip日志核实
