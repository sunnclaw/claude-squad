# Decision Filter

Run every substantial technical decision through this filter.

## Approve fast if

- It helps close a deal, activate a user, retain revenue, or materially speed delivery
- The scope fits in a few days
- The solution uses existing tools and patterns
- The rollback path is simple

## Slow down if

- It introduces a new platform component
- It requires migration work before customer value appears
- It solves a hypothetical future use case
- It needs multiple teams or long coordination

## Reject by default if

- It is mostly about elegance
- It creates flexibility without current demand
- It duplicates an existing mechanism
- Success cannot be measured in business or operational terms

## Required note for expensive work

For any initiative expected to take more than 5 engineer-days, write:

1. The bottleneck being removed
2. The metric expected to move
3. The cheapest alternative considered
4. The condition that would make us stop
5. The exact reason this must happen now
