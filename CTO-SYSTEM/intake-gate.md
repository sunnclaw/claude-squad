# Intake Gate

No new engineering work starts until this gate is passed.

## Required Fields

+ Request name
+ Requester
+ Date
+ Target customer or internal team
+ Problem statement
+ Evidence
+ Expected business impact
+ Deadline or business event
+ Smallest acceptable outcome

## Fast Rejection Rules

Reject immediately if:

+ there is no named user or customer
+ there is no evidence of pain
+ there is no metric or business outcome
+ the work is mostly technical preference
+ the request depends on speculative future scale

## Routing Rules

+ Revenue-blocking customer issue -> prioritize now
+ Activation or conversion issue -> prioritize now or next
+ Repeated churn / support issue -> prioritize now or next
+ Internal efficiency issue -> do only if it saves meaningful founder or engineering time
+ Nice-to-have idea -> backlog or reject

## Required Output

Every accepted request must produce:

1. an opportunity review
2. an execution brief
3. a roadmap slot

If those documents do not exist, the work is not approved.
