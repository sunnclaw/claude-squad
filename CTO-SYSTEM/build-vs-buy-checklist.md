# Build vs Buy Checklist

Default answer: buy or use a manual workaround.

Build only when several of these are true:

+ the capability is core to the product's differentiation
+ the workflow is frequent and painful
+ off-the-shelf options fail critical requirements
+ integration cost is lower than ongoing vendor cost and constraints
+ owning the system creates compounding strategic advantage

Buy when several of these are true:

+ the function is commodity
+ speed matters more than customization
+ demand is still uncertain
+ there is a reliable vendor with acceptable switching cost
+ the feature is internal-facing and not differentiating

Use manual work when several of these are true:

+ task volume is still low
+ the workflow is changing every week
+ the process is not yet understood
+ automation would lock in assumptions too early

## Required Questions

1. Is this core to what customers pay us for?
2. What happens if we do nothing for 30 days?
3. What is the manual workaround?
4. What is the cheapest acceptable vendor option?
5. What evidence says we will need this six months from now?

If these cannot be answered clearly, do not build.
