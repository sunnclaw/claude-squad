# Overbuilding Filters

Apply this list during planning and review.

Reject or re-scope the work if:

+ the proposal starts with architecture instead of customer pain
+ the first release takes more than one week to deliver value
+ the design optimizes for hypothetical future scale
+ the solution requires a platform layer for only one use case
+ the team cannot name the metric it should move
+ the work introduces multiple new technologies without a direct need
+ the proposed flexibility is not required by today's customers
+ the problem can be handled manually for now
+ no kill criteria are defined

## Green-Light Shape

Healthy startup engineering work usually looks like this:

+ one painful problem
+ one clear owner
+ one measurable outcome
+ one small release
+ one short feedback loop
