# Operating System

Start here:

1. Read `cto-system.md` and adopt its constraints as default policy.
2. Use `templates/opportunity-brief.md` before starting any material build.
3. Use `templates/weekly-review.md` every Friday to keep engineering tied to revenue.

Suggested operating loop:

1. Pick one revenue constraint.
2. Write an opportunity brief.
3. Ship the smallest version in 1 week.
4. Review the scoreboard.
5. Cut or double down based on evidence.
