# Build vs Buy Checklist

Default answer: buy or use an existing service.

Build only if at least one of these is true:

- the capability is core to product differentiation
- vendor pricing destroys unit economics at expected scale
- compliance or data constraints rule vendors out
- integration requirements are unique and proven
- a manual process has already repeated enough times to justify automation

Do not build if most of these are true:

- the team wants control more than outcomes
- the problem is back-office, not customer-facing
- there is no measured pain yet
- a vendor solves at least 80% of the problem
- custom code would create ongoing maintenance with no revenue upside

Required before approving internal build:

1. Name the vendor options reviewed.
2. Estimate 12-month total cost of ownership for build and buy.
3. State the customer or margin advantage from building.
4. Define the kill criteria if the build does not pay off.
5. Define the temporary manual workaround.
