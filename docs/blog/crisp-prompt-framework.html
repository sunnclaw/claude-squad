什么是CRISP框架？

CRISP是五个关键要素的首字母缩写：

  • Context（背景）- 提供必要的上下文
  • Role（角色）- 定义AI的角色定位
  • Instruction（指令）- 明确具体的任务
  • Specifics（细节）- 补充关键细节
  • Parameters（参数）- 设定输出格式和限制

C - Context（背景）

核心原则：AI需要知道"为什么"才能给出"是什么"。

背景信息帮助AI理解你的处境、目标和约束。没有背景，AI只能给出通用答案。

背景信息应包含：

  • 你当前面临的问题或挑战
  • 你的目标是什么
  • 相关的背景知识或前情
  • 时间、资源等约束条件

R - Role（角色）

核心原则：不同的角色看到不同的世界。

给AI设定一个专业角色，它会调用该领域的专业知识和表达方式。

常用角色设定：

  • 资深产品经理 - 关注用户价值和商业价值
  • 技术架构师 - 关注系统设计和扩展性
  • 内容营销专家 - 关注传播和转化
  • 数据分析师 - 关注数据洞察和决策支持

I - Instruction（指令）

核心原则：指令越具体，执行越精准。

明确告诉AI你要它做什么。使用动词开头，避免模糊表述。

好的指令特点：

  • 使用明确的动词：分析、撰写、设计、评估
  • 说明期望的深度：简要概述 vs 详细分析
  • 界定范围：包含什么、不包含什么

S - Specifics（细节）

核心原则：细节决定成败。

补充任何可能影响输出质量的具体信息。

常见的细节补充：

  • 目标受众：文章写给谁看
  • 风格要求：正式/轻松、专业/通俗
  • 参考资料：可以参考哪些资料
  • 注意事项：需要避免什么

P - Parameters（参数）

核心原则：格式约束让输出更可用。

设定输出的格式、长度、结构等参数，让结果更符合你的需求。

常见参数设定：

  • 字数限制：200字以内 / 1000字左右
  • 格式要求：列表、表格、段落
  • 结构要求：分几个部分、每部分包含什么
  • 语言风格：中文、英文、中英混合

实战对比

❌ 普通提示词

"帮我写一篇关于AI的文章"

结果：泛泛而谈的AI介绍，缺乏深度和针对性

✓ CRISP提示词

"我是一名科技自媒体博主，目标读者是对AI感兴趣但不太懂技术的职场人。请以科技评论员的身份，撰写一篇800字左右的文章，主题是'普通人如何在工作中用好AI工具'。要求：用通俗的语言解释AI工具的使用场景，给出3个具体可操作的建议，风格轻松但专业。"

结果：有针对性、可操作性强、符合目标读者需求

CRISP模板

通用CRISP模板

【背景】我正在...（描述你的处境）

【角色】请你扮演一位...（定义AI角色）

【指令】帮我...（明确任务）

【细节】需要注意...（补充细节）

【参数】输出格式为...（设定参数）

进阶技巧

1. 迭代优化

第一次输出后，根据结果追问和调整。CRISP不是一次性框架，而是迭代优化的工具。

2. 组合使用

复杂任务可以拆解成多个CRISP提示词，分步骤执行。

3. 保存模板

将常用的CRISP提示词保存为模板，提高复用效率。

常见误区

  • 误区1：提示词越长越好 → 实际上，关键是结构清晰，不是字数多
  • 误区2：一次就要完美 → 实际上，迭代优化比一次完美更高效
  • 误区3：照搬别人的提示词 → 实际上，要根据自己需求调整
