1. 简介：为什么需要AI代理团队

单一AI模型虽然强大，但在复杂业务场景下往往力不从心。想象一下，你需要同时处理客户咨询、内容创作、代码审查、数据分析——让一个AI同时做好所有这些事情几乎不可能。

这就是AI代理团队的价值：每个代理专注于自己擅长的领域，通过协作完成复杂任务。就像现实中的公司有CEO、工程师、设计师、客服一样，AI代理团队也是这样组织的。

Paperclip是什么？

Paperclip是一个AI代理协作管理框架，它提供了：

  • 代理管理：注册、配置、监控AI代理
  • 任务分配：基于Issue的任务流转机制
  • 状态追踪：实时了解每个代理的工作状态
  • API接口：标准化的REST API，易于集成

2. 准备工作

前提条件

  • Node.js 18+ 环境
  • 基础的命令行操作能力
  • 对AI和LLM的基本了解

首先，确保你的系统已安装Node.js和npm：

3. 理解Paperclip架构

核心概念

概念 说明
Control Plane 控制中心，处理代理注册、任务分配、状态管理
Agent AI代理，有特定角色和能力，执行具体任务
Issue 任务单元，包含描述、状态、分配信息
Company 组织单位，一个公司可以有多个代理

系统架构图

┌─────────────────────────────────────────────────────────────────┐ │ Paperclip Control Plane │ │ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ │ │ Agent │ │ Issue │ │ Company │ │ │ │ Service │ │ Service │ │ Service │ │ │ └─────────────┘ └─────────────┘ └─────────────┘ │ │ │ │ │ REST API │ └─────────────────────────┬───────────────────────────────────────┘ │ ┌────────────────┼────────────────┐ │ │ │ ▼ ▼ ▼ ┌──────────┐ ┌──────────┐ ┌──────────┐ │ CEO │ │ Engineer │ │ Analyst │ │ Agent │ │ Agent │ │ Agent │ └──────────┘ └──────────┘ └──────────┘

数据流

  1. 任务创建：CEO代理或外部系统创建Issue
  2. 任务分配：Issue被分配给特定代理
  3. 代理认领：代理调用checkout API认领任务
  4. 执行任务：代理根据Issue描述执行工作
  5. 状态更新：完成后更新Issue状态为done

4. 创建第一个代理

步骤1：创建配置文件

每个代理都需要一个AGENTS.md配置文件。让我们创建一个工程师代理：

文件：agents/engineer/AGENTS.md

# 工程师代理 - Claude天团 你是 Claude天团 的工程师，负责技术实现和代码开发。 ## ⚠️ 强制规则 - 必须遵守 ### 规则 1: 只做分配的任务 你**只能**做分配给你的 Issue 任务。**禁止**做任何其他工作。 ### 规则 2: 必须遵循 Paperclip 流程 每次心跳必须执行以下步骤： 1. 调用 paperclip skill 2. GET /api/agents/me 获取身份 3. GET /api/companies/{companyId}/issues?assigneeAgentId={yourId} 4. 如果有任务，POST /api/issues/{issueId}/checkout 认领 5. 执行任务 6. PATCH /api/issues/{issueId} 更新状态 ## 工作流程 ### 技术实现任务 1. 阅读Issue描述，理解需求 2. Checkout这个Issue 3. 编写代码/文档 4. 测试验证 5. 更新Issue状态为done ## 环境变量（自动注入） - PAPERCLIP_AGENT_ID: your-agent-id - PAPERCLIP_COMPANY_ID: your-company-id - PAPERCLIP_API_URL: http://127.0.0.1:3100/api ## 公司信息 - 公司名称: Claude天团 - 工作目录: /path/to/work

步骤2：注册代理

向Control Plane注册你的代理：

# 使用API注册代理 curl -X POST http://127.0.0.1:3100/api/agents \ -H "Content-Type: application/json" \ -d '{ "name": "Founding Engineer", "type": "engineer", "companyId": "7f75017f-bf26-46e4-bd97-cb508f831442", "config": { "model": "claude-sonnet-4-6", "maxTokens": 8192 } }' # 响应示例 { "id": "27344c6d-cd4a-4fc6-b80c-eb18047eb3f7", "name": "Founding Engineer", "type": "engineer", "status": "idle" }
提示：保存返回的id，这是代理的身份标识，后续所有操作都需要使用。

步骤3：启动代理

使用AI运行时启动你的代理：

# 设置环境变量 export PAPERCLIP_AGENT_ID=27344c6d-cd4a-4fc6-b80c-eb18047eb3f7 export PAPERCLIP_COMPANY_ID=7f75017f-bf26-46e4-bd97-cb508f831442 export PAPERCLIP_API_URL=http://127.0.0.1:3100/api # 启动Claude Code claude-code --agent-file agents/engineer/AGENTS.md

5. 设计代理角色

常见代理类型

类型 职责 适合任务
CEO 决策、协调、质量把控 订单确认、任务分配、最终审核
Engineer 技术实现 代码开发、Bug修复、技术文档
Analyst 分析研究 市场调研、数据分析、研究报告
Writer 内容创作 博客文章、营销文案、用户手册

提示词设计原则

好的提示词应该：
  • 明确边界：清楚定义代理能做什么、不能做什么
  • 结构化指令：使用编号步骤而非模糊描述
  • 提供示例：在配置中包含典型工作流程示例
  • 强制规则优先：关键规则放在最前面

CEO代理示例

# CEO代理 - Claude天团 你是 Claude天团 的CEO，负责公司整体运营。 ## ⚠️ 强制规则 ### 规则 1: 订单确认 收到新订单后，必须： 1. 审核订单内容是否完整 2. 确认报价是否合理 3. 分配给合适的代理 ### 规则 2: 质量把控 所有交付物必须经过你的审核才能发送给客户。 ## 工作流程 ### 订单处理 1. 接收订单通知 2. 评估工作量和报价 3. 创建Issue分配给相应代理 4. 跟踪进度 5. 审核交付物 6. 发送给客户 ## 决策框架 ### 报价策略 - 内容创作：¥100/千字 - 代码审查：¥50/文件 - 研究报告：¥200/主题 - 加急服务：+50%

6. 任务流转机制

Issue状态流转

┌─────────┐ │ todo │ ← 新创建的任务 └────┬────┘ │ checkout ▼ ┌───────────────┐ │ in_progress │ ← 代理正在处理 └───────┬───────┘ ┌────┴────┐ │ │ ▼ ▼ ┌─────────┐ ┌──────────┐ │ done │ │ blocked │ └─────────┘ └────┬─────┘ │ unblock ▼ ┌───────────────┐ │ in_progress │ └───────────────┘

API调用流程

// 1. 获取身份验证 GET /api/agents/me Headers: { "x-agent-id": "your-agent-id" } // 2. 查询待处理任务 GET /api/companies/{companyId}/issues?assigneeAgentId={myId}&status=todo // 3. 认领任务 POST /api/issues/{issueId}/checkout // 4. 执行任务... // (实际工作代码) // 5. 标记完成 PATCH /api/issues/{issueId} Body: { "status": "done", "comment": "任务完成说明..." }

处理阻塞情况

当代理遇到无法独立解决的问题时，可以将任务标记为blocked：

PATCH /api/issues/{issueId} { "status": "blocked", "comment": "需要CEO确认技术方案后才能继续。等待中..." }
注意：标记为blocked的任务会在控制台显示警告，CEO代理会收到通知。记得在问题解决后将状态改回in_progress。

7. 高级技巧

多代理协作

对于复杂任务，可以让多个代理协作完成：

示例：完整内容项目
  1. CEO：接收订单，拆分为多个Issue
  2. Analyst：研究主题，输出大纲
  3. Writer：根据大纲撰写内容
  4. Editor：审核修改，确保质量
  5. CEO：最终审核，发送客户

心跳机制

代理应定期发送心跳以保持在线状态：

// 每30-60秒更新一次状态 setInterval(async () => { await fetch(`${API_URL}/agents/${AGENT_ID}`, { method: 'PATCH', headers: { 'x-agent-id': AGENT_ID, 'Content-Type': 'application/json' }, body: JSON.stringify({ status: currentStatus, // 'idle' or 'working' lastHeartbeat: new Date().toISOString() }) }); }, 30000); // 30秒

错误处理

错误码 原因 处理方式
401 Agent ID无效 检查环境变量配置
404 Issue不存在 重新查询Issue列表
409 Issue已被认领 查询下一个可用Issue
429 请求频率过高 实现指数退避重试

8. 最佳实践

提示词优化

  • 使用具体的指令而非模糊描述
  • 提供工作示例帮助代理理解预期输出
  • 设置边界条件，明确什么情况下应该寻求帮助
  • 定期更新提示词，基于实际运行效果优化

监控和调试

  • 使用Dashboard查看代理状态和任务进度
  • 检查Issue的comment历史了解工作过程
  • 设置告警监控代理离线或任务超时
  • 保留执行日志便于问题追溯

安全考虑

安全要点：
  • Agent ID通过环境变量注入，不要硬编码
  • 限制代理的文件访问权限
  • 敏感操作需要额外授权
  • 定期轮换凭证
