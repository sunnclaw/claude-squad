# 90-Day Execution Plan

## Phase 1: Days 1-30

Goal: tighten the revenue loop

- instrument activation, conversion, and churn events
- identify the single biggest onboarding or sales blocker
- ship weekly fixes to the top blocker
- remove or pause non-revenue projects
- create a simple customer feedback loop from sales and support

Exit criteria:

- weekly scoreboard is updated consistently
- one bottleneck is named each week
- cycle time from idea to production is visible

## Phase 2: Days 31-60

Goal: improve retention and conversion through the core workflow

- rank the top 5 user pain points from real usage
- ship only improvements tied to activation or repeat usage
- tighten billing and payment reliability
- standardize incident handling for revenue-path issues

Exit criteria:

- the team can explain the main drop-off point in the funnel
- at least 2 shipped changes are tied to measured customer behavior

## Phase 3: Days 61-90

Goal: add leverage without losing focus

- automate only the manual workflows repeated enough to justify it
- add missing internal tools only where they shorten cycle time
- improve observability around the highest-cost production issues

Exit criteria:

- internal tooling work is a minority of total engineering effort
- leverage work clearly supports revenue or retention work

## Rules Across All 90 Days

- no rewrite
- no platform project without direct business justification
- no more than 3 active priorities
- every Friday ends with keep, expand, or cut decisions
