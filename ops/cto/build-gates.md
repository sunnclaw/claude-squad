# Build Gates

Run these gates before approving, designing, or expanding any work.

## Gate 1: Revenue Link

Can someone explain in one sentence how this helps:

- win a deal,
- activate a user,
- retain a customer, or
- reduce cost on current revenue delivery?

If not, do not start.

## Gate 2: Constraint Match

Does this attack the current top bottleneck from the scorecard?

If not, delay it.

## Gate 3: Smallest Sellable Slice

What is the smallest version that can be shipped in 1 week or less?

If the proposal cannot be sliced, the design is too big.

## Gate 4: Manual First

Can operations, a script, or a narrow workflow solve this first?

If yes, do that before building a generalized system.

## Gate 5: Named Tradeoff

What are we explicitly not building right now?

Every initiative must include a refusal list.

## Gate 6: Metric and Owner

Who owns the result, and what metric changes if this works?

If there is no owner or no metric, stop.

## Gate 7: Kill Condition

What result would tell us to stop, roll back, or avoid phase 2?

If there is no kill condition, the work will sprawl.

## Automatic Rejections

Reject by default when proposals include:

- "platform"
- "framework"
- "flexible for future use cases"
- "we might need this later"
- "let's redesign it properly first"
- "let's build admin tooling before validating demand"

These are not always wrong. They are wrong until proven necessary by current demand.
