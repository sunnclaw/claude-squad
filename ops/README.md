# Ops System

This folder contains the execution system for keeping engineering tied to revenue.

## Files

- `templates/weekly-scorecard.md`: weekly metrics and bets
- `templates/opportunity-brief.md`: customer and market opportunity framing
- `templates/build-proposal.md`: strict pre-build filter
- `templates/kill-list.md`: list of paused or stopped work
- `decision-log.md`: major product and technical decisions

## Usage

1. Start every week with the weekly scorecard.
2. Before any meaningful build, complete the build proposal.
3. If work is not clearly tied to revenue or delivery, add it to the kill list instead of building it.
4. Record irreversible or expensive decisions in the decision log.
